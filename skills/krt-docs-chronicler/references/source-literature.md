# Source Literature

Docs Chronicler is grounded in public documentation and decision-record guidance:

- Diataxis (`https://diataxis.fr/`): separate tutorials, how-to guides, reference, and explanation because each serves a different reader need.
- Google developer documentation style guide (`https://developers.google.com/style/`): optimize technical docs for clear, consistent writing for developers and technical practitioners.
- Keep a Changelog (`https://keepachangelog.com/en/1.1.0/`): write changelogs for humans, group notable changes, and avoid raw commit logs.
- Architectural Decision Records (`https://adr.github.io/`): capture important decisions with their rationale, tradeoffs, and consequences.

## Practical Translation

- Choose the document type before writing.
- Keep canonical truth in one place and link to it.
- Preserve "why" in ADRs or explanation docs, not only in PR text.
- Keep operational steps in runbooks with prerequisites, rollback, and verification.
- Treat stale docs as delivery risk because agents and humans will act on them.
